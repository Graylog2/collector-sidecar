Graylog Collector
=================

## Build

The v2 build is in a transition period: both `make` and `go-task` produce the same artifacts.
The Taskfile is the canonical source going forward; the Makefile v2 targets remain temporarily
for validation and will be removed once the migration is complete.

Build the host-platform `./graylog-collector` binary:

```
go tool task v2:build       # via go-task (preferred)
make v2                     # via Make (transitional)
```

Cross-compile for all five platforms (linux/amd64, linux/arm64, darwin/amd64, darwin/arm64,
windows/amd64):

```
go tool task v2:all
make v2all
```

Run all tests:

```
go tool task v2:test
make v2test
```

List available tasks:

```
go tool task -l
```

### Why both?

The Makefile remains for the v1 sidecar build, which has not been migrated. During the v2
transition, v2 targets are duplicated between `Makefile` and `Taskfile.yml` so we can verify
artifact equivalence. When v1 retires, both will be cleaned up.

## Run

- `--dev` flag sets data path to `./data`
- `--sidecar` enables legacy Sidecar extension
- `--server-url` takes WebSocket or HTTP server URL (default: `ws://localhost:9000/v1/opamp`
- `--token` bearer token

```
./graylog-collector supervisor --dev

./graylog-collector supervisor --dev --sidecar

./graylog-collector supervisor --dev --server-url ws://localhost:1234/custom/path

./graylog-collector supervisor --dev --token s3cr3t --server-url https://localhost:1234/custom/path
```
